# QA Checklist

## 1. Tone & Content Principles
- [x] **Không khuyến nghị mua/bán:** Kiểm tra các bài Business, đảm bảo disclaimer rõ ràng.
- [x] **Không FOMO:** Không dùng từ ngữ "kèo", "hàng nóng", "full margin".
- [x] **Trung thực:** Luôn có mục Risks trong bài phân tích.
- [x] **Calm UI:** Màu sắc trung tính (Stone/Gray), không đỏ xanh lòe loẹt.

## 2. UX & Reading Experience
- [x] **Focus Mode:** Toggle hoạt động, ẩn header/sidebar.
- [x] **Typography:** Font Inter dễ đọc, line-height thoáng.
- [x] **Settings:**
  - [x] Font Size (Normal / Large)
  - [x] Line Width (Normal / Wide)
- [x] **Reading Progress:** Thanh tiến trình mảnh trên cùng.

## 3. Technical & SEO
- [x] **Mobile Responsive:** Layout hiển thị tốt trên mobile.
- [x] **Dark Mode:** Hoạt động tốt, contrast đủ nhưng không chói.
- [x] **Metadata:** Các trang [slug] đều có dynamic title/description.
- [x] **Semantic HTML:** Sử dụng đúng thẻ h1, h2, article, time.

## 4. Business Analysis Logic
- [x] **Logic 10 bước:** Template MDX bắt buộc đủ 10 mục.
- [x] **Sleep-well Test:** Kết luận luôn hướng về sự an tâm dài hạn.
- [x] **Risk Level:** Hiển thị rõ ràng (Low/Medium/High).
